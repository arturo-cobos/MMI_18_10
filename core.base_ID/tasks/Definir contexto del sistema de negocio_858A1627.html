Tarea: Definir contexto del sistema
Define como crear un diagrama de contexto del sistema que muestra a alto nivel la colaboración entre el sistema de negocio y sus actores de negocio
Disciplinas: Análisis de Negocio e Ingeniería de Requerimientos
Objetivo
  • Permitir la comprensión de necesidades de un sistema y las asociaciones entre el sistema y sus actores
  • Definir interfaces y relaciones entre actores del sistema
  • Definir la visión y alcance del sistema
Relaciones
RolesRealizador principal:
  • Ingeniero de Requerimientos
Otras actividades adicionales que realiza:
EntradasObligatoria:
  • Alcance y Contexto del Sistema
  • Enfoque del análisis de negocio
  • Modelo de Caso de Uso
    		Opcional:
    • Ninguno
    Salidas
    • Alcance y Contexto del Sistema
      Pasos
      Definir el límite del sistema
      Este separa la parte de la realidad que puede ser modificada o alterada por el proceso de desarrollo de los aspectos del entorno que no pueden ser cambiados o modificados por el proceso de desarrollo
      Definir el límite del contexto
      Separa la parte relevante del sistema a desarrollarse de la parte que no es considerada durante la ingeniería de requerimientos
      Documentar el contexto del sistema
      Mediante un diagrama se abstraen los aspectos que influyen en el contexto para una mayor comprensión
